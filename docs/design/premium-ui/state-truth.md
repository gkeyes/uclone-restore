# UClone UI 状态真相表

## 1. 原则

- 每个状态必须指向一个明确数据源。
- “未知”是合法状态，不以旧缓存或乐观默认替代。
- 颜色只能辅助状态；必须同时使用文字和图标/形状。
- 技术原值作为次级信息，不能成为普通用户唯一可见文案。
- 页面刷新失败时保留数据与保留“已确认状态”是两件事；无法证明的实时状态必须降级。

## 2. App 数据来源状态

### 当前生产映射

| 状态 | 成立条件 | 用户文案 | 主动作 | 不允许的推断 | 等级 |
| --- | --- | --- | --- | --- | --- |
| `MAIN` | 工作区索引读取成功，包不在 UNKNOWN 集合，且没有 switch marker | 主数据正在使用 | 切换到分数据 | 当前代码会把“无 marker”映射为 MAIN，但这不构成数据来源已被正向证明 | `PROJECT-VERIFIED` |
| `CLONE` | user0 已知为分数据，且存在有效 MAIN 返回点 | 分数据正在使用 | 还原主数据 | 不能等同 user10 正在运行 | `PROJECT-VERIFIED` |
| `UNKNOWN` | 工作区读取失败，或包在 UNKNOWN marker 集合中 | 当前数据来源待确认 | 检查状态/进入详情 | 不得直接显示切换或还原为确定动作 | `PROJECT-VERIFIED` |

### 已确认的需求差异

`需求.md` 要求“无法证明的数据来源必须是 UNKNOWN，刷新或显示重置不得制造 MAIN”。当前生产映射仍以“没有 switch marker”作为 MAIN 默认值。这是 `PROJECT-VERIFIED` 的需求/实现差异，不是视觉设计选择。

本设计分支只记录差异：第一批 UI-only 实现继续消费现有 `AppDataState`，不得偷偷改变状态判定；状态逻辑修复必须独立计划、增加回归测试并经真机验证后实施。

## 3. 分身用户生命周期

| 底层状态 | 主文案 | 建议情境动作 | 风险说明 |
| --- | --- | --- | --- |
| 不存在/不可用 | 未找到分身系统 | 查看诊断 | 禁止数据任务 |
| `NotStarted` | 分身系统未启动 | 启动分身 | CE 不可读 |
| `StartedLocked` | 分身系统已启动，数据未解锁 | 解锁分身 | CE 数据任务不能继续 |
| `RunningUnlocked` | 分身系统已就绪 | 无主动作；可进入管理 | 可以读取 CE，不等于目标 App 已安装 |
| `Unknown`/查询失败 | 分身状态读取失败 | 重新检测 | 不能缓存成“已就绪” |

技术原值如 `RUNNING_UNLOCKED`、user10 只在展开详情或诊断中显示。

## 4. 任务状态

| `TaskStatus` | 用户文案 | UI 模式 | 是否终态 | 主要操作 |
| --- | --- | --- | --- | --- |
| `ACCEPTED` | 已接收，等待执行 | 轻量进行中 | 否 | 查看详情 |
| `RUNNING` | 正在执行 | 当前以阶段/步骤为主 | 否 | 查看日志；仅在安全能力存在时显示取消 |
| `AUTO_ROLLING_BACK` | 正在自动回滚 | 高优先级警示 | 否 | 禁止冲突动作 |
| `ROLLED_BACK` | 操作未完成，已恢复操作前数据 | 警告终态 | 是 | 查看原因与日志 |
| `SUCCESS` | 已完成 | 成功终态 | 是 | 查看结果 |
| `SUCCESS_WITH_WARNINGS` | 已完成，部分项目未迁移 | 警告终态 | 是 | 查看警告明细 |
| `FAILED` | 未完成 | 失败终态 | 是 | 查看失败阶段和可执行下一步 |
| `FAILED_FATAL` | 严重失败，需要人工检查 | 危险终态 | 是 | 进入诊断；禁止用普通成功样式 |
| `INTERRUPTED` | 任务被中断 | 中性/警告终态 | 是 | 检查目标状态后重试 |

## 5. 任务阶段

| 阶段 | 面向用户文案 | 关键状态信息 |
| --- | --- | --- |
| `PRECHECK` | 检查运行条件 | Root、用户、安装、空间 |
| `SOURCE_PREPARE` | 准备源数据 | 数据范围、扫描/复制数量 |
| `TARGET_STOP` | 停止目标 App | 目标用户与包 |
| `ROLLBACK_BACKUP` | 生成本次操作前保护 | 备份路径、数据量 |
| `RESTORE_DATA` | 写入目标数据 | 已处理文件/字节 |
| `RESTORE_METADATA` | 修复文件属性 | UID/GID/SELinux 结果 |
| `RESTORE_PERMISSIONS` | 迁移权限与 AppOps | 成功/跳过/警告数 |
| `VERIFY` | 验证结果 | 完整性与目标检查 |
| `COMMIT` | 提交任务结果 | marker/回滚提交 |
| `AUTO_ROLLBACK` | 自动回滚 | 回滚结果必须显著 |
| `CLEANUP` | 清理临时数据 | 不遮蔽主结果 |

进度百分比只有在分母可信时显示；否则展示“第 n/m 阶段”和可验证的文件/字节信息。

当前 `TaskProgress` 只暴露任务、步骤和实时日志，没有稳定的实时总文件/总字节分母。当前实现因此只能可靠展示阶段/步骤；确定百分比属于需要单独遥测契约的候选能力，不得在纯 UI 改造中推算。

## 6. 外部请求状态

外部请求回执与 `TaskStatus` 是两套不同状态，不得混在同一枚举或视觉标签中。

| 外部状态 | 当前产生位置 | 用户语义 | 当前证据 |
| --- | --- | --- | --- |
| `ACCEPTED` | Service 完成校验并获得 `TaskCoordinator` admission 后 | 请求已进入业务任务 | `PROJECT-VERIFIED` |
| `SUCCESS` | 已接受任务以 `SUCCESS` 或 `SUCCESS_WITH_WARNINGS` 结束 | 外部调用完成；警告详情仍来自任务记录 | `PROJECT-VERIFIED` |
| `FAILED` | 前台服务启动失败、执行异常或业务任务非成功终态 | 请求或任务未完成 | `PROJECT-VERIFIED` |
| `REJECTED` | Intent/协议/策略校验失败 | 请求未进入任务层 | `PROJECT-VERIFIED` |
| `BUSY` | 另一个 requestId 已占用协调器 | 当前请求未执行 | `PROJECT-VERIFIED` |
| `ALREADY_RUNNING` | 相同 requestId 已在执行 | 不重复提交 | `PROJECT-VERIFIED` |
| `NEED_CONFIRMATION` | 仅在 contract 声明 | 当前生产代码没有发出路径 | `PROJECT-VERIFIED` |
| `NEED_USER_ACTION` | 仅在 contract 声明 | 当前生产代码没有发出路径 | `PROJECT-VERIFIED` |

当前外部协议没有单独广播 `RUNNING`、`SUCCESS_WITH_WARNINGS` 或 `INTERRUPTED`。模块可收到 `ACCEPTED` 和终态回执，但更细的业务状态应从任务记录读取，设计稿不得虚构传输阶段。

## 7. 安装状态

| user0 | user10 | UI 状态 | 可用动作 |
| --- | --- | --- | --- |
| 已安装 | 未安装 | 仅主系统已安装 | 安装到分身；安装并迁移权限；安装并同步数据 |
| 未安装 | 已安装 | 仅分身系统已安装 | 安装到主系统；对应组合动作 |
| 已安装 | 已安装 | 两侧均已安装 | 数据动作；不显示重复安装为主动作 |
| 未安装 | 未安装 | 包不可用 | 不提供跨用户安装 |
| 查询失败 | 任意 | 安装状态未知 | 重新检查，不执行覆盖性组合动作 |

安装并同步是分阶段结果：安装成功、权限迁移警告、数据同步失败可同时成立。

## 8. 工作区归属维护状态

| 状态 | 来源 | UI 文案 | 动作 |
| --- | --- | --- | --- |
| 未扫描 | `workspaceOwnership == null` | 尚未扫描备份归属 | 只读扫描 |
| 正常 | `nonRootEntries == 0` 且扫描契约匹配当前设置 | 未发现异常归属 | 可重新扫描 |
| 异常 | `nonRootEntries > 0` 且扫描契约匹配 | 发现 n 个非 root 归属项 | 二次确认后修复 |
| 过期 | rootDir/userId 与扫描报告不一致 | 扫描结果已过期 | 必须重新扫描；禁止修复 |
| 扫描/修复中 | 正式任务状态 | 正在扫描/修复 | 禁止冲突数据任务 |
| 中断 | 任务终态 | 修复中断，可安全重新执行 | 重新扫描后重试 |

## 9. 候选页面加载与错误状态

以下是目标视觉契约，不是当前 `UiState` 已实现的数据合同。当前状态模型只有全局 `busy`、`message`、可空 `environment` 和各列表，没有逐页 loading/error/stale 字段。除现有字段能直接证明的状态外，以下能力均为 `EXPERIMENTAL`，需要独立状态模型任务和用户批准，不能夹带进第一批 UI-only 提交。

| 页面 | 候选加载 | 当前可证明空状态 | 候选错误状态 | 旧数据策略 | 等级 |
| --- | --- | --- | --- | --- | --- |
| 首页 | 状态占位 + 当前任务 | 无收藏 App | 环境读取失败 | 实时环境不能沿用为已确认状态 | `EXPERIMENTAL` |
| App | 列表加载反馈 | 没有匹配 App | 包列表读取失败 | 只有新增 freshness 字段后才能显示旧列表 | `EXPERIMENTAL` |
| 数据 | 工作区索引加载反馈 | 尚无备份 | 工作区读取失败 | 当前解析失败会返回 `readSucceeded=false`；执行禁用需单独核对调用链 | `EXPERIMENTAL` |
| 历史 | 轻量加载反馈 | 尚无任务记录 | 历史读取失败 | 当前没有缓存时间字段 | `EXPERIMENTAL` |
| 设置 | 表单加载反馈 | 不适用 | 保存/验证失败 | 当前草稿保留行为可复用，字段错误需新状态 | `EXPERIMENTAL` |
| 诊断 | 正在检测 | 无可用结果 | 命令失败 | 现有可空环境可显示未检测，不等于完整错误模型 | `EXPERIMENTAL` |
| 模块 App | 包列表加载反馈 | 无候选 App | PackageManager 读取失败 | 当前没有显式 stale 模型 | `EXPERIMENTAL` |
| 模块日志 | 日志加载反馈 | 暂无日志 | 日志读取失败 | 不把空日志推导为实时健康 | `EXPERIMENTAL` |

## 10. 可访问性状态表达

- 成功：勾选图形 + “已完成/正常”。
- 警告：警告图形 + 具体未完成内容。
- 失败：错误图形 + 失败阶段 + 下一步。
- 运行：进度指示 + 阶段文字 + 已用时间。
- 禁用：降低视觉强调的同时保留可读原因；TalkBack 描述“不可用，原因……”。
- 风险：高风险/系统 App 以标签和说明呈现，不仅依靠红色。
