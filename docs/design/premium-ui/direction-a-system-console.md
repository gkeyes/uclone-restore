# 方向 A：系统控制台

- 推荐等级：优先推荐
- 审批状态：`EXPERIMENTAL`
- 功能变化：不删除、不移动、不新增现有功能
- 实施风险：低至中

## 1. Visual thesis

UClone 是一个高风险、长任务、跨用户的系统工具。方向 A 把它设计成“安静但可审计的系统控制台”：用户先看到系统是否可操作，再看到当前任务，最后进入 App 和数据工具。界面通过稳定的列表节奏、紧凑状态行、明确的主次动作建立信任，而不是通过大面积玻璃、渐变或连续卡片制造视觉冲击。

产品承诺：

> 每次操作前知道当前状态、将要覆盖什么、保护在哪里；操作后知道结果、警告和下一步。

## 2. Composition

### Compact phone

```text
┌──────────────────────────────┐
│ ☰  首页               任务状态 │  TopAppBar
├──────────────────────────────┤
│ 系统就绪                       │  Health summary
│ Root 正常 · 分身已解锁          │
│ [查看诊断]                     │
├──────────────────────────────┤
│ 当前任务                       │
│ 小红书  写入分身数据             │
│ 第 5/11 阶段                    │
├──────────────────────────────┤
│ 收藏 App                      │
│ [图标] 小红书   分数据正在使用  │
│        user0 · 可还原     [还原] │
│ [图标] Chrome  主数据正在使用   │
│        user0             [切换] │
└──────────────────────────────┘
```

- 6 个现有顶层页面全部保留为同级入口，放入 modal navigation drawer。
- Top app bar 显示当前页面；右侧只承载当前任务/全局状态入口。
- 内容用全宽 section 与列表行，不把每个 section 做成悬浮卡片。
- 首页只显示情境动作；完整生命周期管理仍保留在现有页面/入口，不删除。

### Medium / expanded

```text
┌────────────┬──────────────────┬──────────────────┐
│ Navigation │ 主内容列表        │ 详情 / 任务检查器   │
│ 首页        │ 系统状态          │ 当前阶段           │
│ App         │ 收藏 App         │ 文件/字节/警告      │
│ 数据        │                  │                  │
│ 历史        │                  │                  │
│ 设置        │                  │                  │
│ 诊断        │                  │                  │
└────────────┴──────────────────┴──────────────────┘
```

- medium 使用 navigation rail；expanded 使用 permanent drawer。
- App 列表、数据列表和历史可采用 list-detail pane。
- 大屏增加信息并列，不简单放大卡片和字号。

## 3. Navigation

| Window | 组件方向 | 功能层级 |
| --- | --- | --- |
| Compact | Modal navigation drawer + TopAppBar | 6 个现有顶层入口保持同级 |
| Medium | Navigation rail | 6 个入口保持同级；详情占第二 pane |
| Expanded | Permanent drawer + list/detail | 6 个入口保持同级；任务可作为第三 pane |

`EXPERIMENTAL`：具体使用 `NavigationSuiteScaffold` 还是显式 drawer/rail，需要实现前根据当前 Material 3 Adaptive 依赖版本确定；不因此新增依赖。

## 4. Information density

- 密度：中高，但不拥挤。
- 每个列表行包含：名称、友好状态、必要技术值、一个行尾主动作。
- 详细解释默认折叠到辅助文字或详情页。
- 状态数字和技术值使用 tabular 视觉节奏，但仍采用 Android 系统字体。
- 主 App 保留现有任务结果与日志路径，不新增打开、复制或导出动作；Launcher 模块日志继续保留当前已有的刷新、复制和清理能力。

## 5. Typography voice

- 页面标题：短、直接、无营销口吻。
- Section 标题：系统状态、当前任务、收藏 App、数据范围、维护工具。
- 行标题：动作对象优先，例如“还原主数据”“更新分身数据”。
- 次级文案：方向、目标用户、风险或技术原值。
- 任务数字：文件、字节、耗时使用统一数字样式；不使用超大 hero 数字。

## 6. Color and material

- 背景采用 Android semantic surface roles，提供 light/dark。
- 主强调色仅用于当前选中导航、单一主动作和确定的交互元素。
- 成功、警告、失败使用语义色 + 图标 + 文案。
- 半透明/材质只限 top bar、drawer/rail 的 chrome；内容面保持不透明和高对比。
- 普通容器圆角不超过 8dp；避免卡片套卡片。
- 禁止渐变背景、装饰光斑、玻璃胶囊按钮堆叠。

## 7. Action hierarchy

| 场景 | 主动作 | 次级动作 | 危险动作 |
| --- | --- | --- | --- |
| 收藏 App | 根据 MAIN/CLONE 显示切换或还原 | 进入详情 | 不在首页显示删除 |
| App 详情 | 当前状态对应动作 | 快照/安装/审计工具行 | 删除主动快照独立危险区 |
| 数据详情 | 恢复主动快照或主系统被动备份 | 查看元数据/日志 | 删除对应主动/被动备份；分身回滚保持列表行内恢复 |
| 设置维护 | 扫描工作区归属 | 修复仅在扫描异常后出现 | 重置工作区独立危险区 |
| 诊断 | 重新检测 | CE/分身调试、解锁、启动与切换 | 关闭分身继续保留在首页，不暗示诊断页已有该动作 |

## 8. Motion

- 页面切换只使用平台默认短过渡。
- 任务阶段变化使用局部内容更新，不用整页闪烁。
- 当前任务以阶段/步骤为主；只有未来提供可信实时分母后，确定进度才平滑更新。
- 错误/警告不使用抖动或强闪烁。
- 系统减少动画时取消非必要的位移和容器变形。

## 9. Main App screen translation

| 当前页面 | 方向 A 结构 |
| --- | --- |
| 首页 | 系统状态摘要 -> 当前任务 -> 收藏 App；生命周期动作按情境出现 |
| App | 搜索/筛选工具栏 -> 高密度 App 行 -> 详情 pane |
| App 详情 | 身份与数据状态 -> 主动作 -> 数据范围 -> 安装/备份/诊断工具行 -> 危险区 |
| 数据 | 三个来源 section，统一 row anatomy，来源标签固定 |
| 历史 | 时间序列列表 + 状态/耗时；宽屏右侧任务详情 |
| 设置 | 分组设置列表；任务专属选项保留在对应说明上下文 |
| 诊断 | 结论优先，原始技术值可展开；工具按只读/生命周期分区 |

## 10. Launcher module translation

- 保留“状态 / App / 控制 / 日志”四个现有顶层页面。
- Compact 使用固定、可滚动的 segmented/tab row 或顶部 page switcher，不再模拟六项底栏。
- 状态页：模块就绪摘要 -> 关键静态检查/历史异常 -> 目标 App 摘要；不宣称实时 Hook/Relay 健康。
- App 页：搜索/筛选 -> 选择列表 -> 固定保存动作。
- 控制页：设置分组行，开关使用 `Switch`，说明使用次级文案。
- 日志页：过滤器、最近日志、复制/清理；清理为危险次级动作。
- 模块仍使用 Android View，不以方向名义迁移框架。

## 11. Official constraints retained

- Android 48dp 最小触控目标。
- Material 3 标准导航、列表、开关、对话框和进度组件。
- 状态不只依靠颜色。
- 大字体下行尾动作允许换行或移到下一行。
- edge-to-edge inset 正确处理。

## 12. Product-specific choices

- MAIN/CLONE/UNKNOWN 是所有 App 行和详情页的第一状态。
- “来源 -> 目标”是高风险确认的固定信息。
- 任务阶段、文件/字节、警告比装饰图更重要。
- 工作区维护与普通设置分区。

## 13. Risks and cost

- 风险：drawer 中入口发现性低于底栏，需要首页和 top bar 提供清晰当前位置。
- 风险：模块顶部四页在大字体下需要滚动或等宽策略。
- 成本：中等，主要涉及主 App 导航壳、页面组件整理、主题和状态组件，不改变后端。
- 回滚：页面内容可逐页迁移；导航壳和主题可作为独立提交回退。

## 14. Approval gate

批准方向 A 仅代表允许后续形成实现 PR，不代表允许删除现有页面、修改 Root 逻辑、引入依赖或发布 APK。
